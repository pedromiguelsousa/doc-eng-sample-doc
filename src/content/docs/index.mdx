This tutorial covers how to create a multi-level navigation bar (navbar) and footer shared between all pages of your website.

![The end result of the tutorial, composed of a sample React application with a navbar and a footer managed in Storyblok.](~/assets/images/tutorial-end-result.png)

After you complete this tutorial, you will be able to create, edit, and delete navbar entries and sub-entries in the Storyblok Visual Editor, as well as configure the entries in your website footer.

The configuration of the navbar and the footer will be defined in a single location so that any changes to these components automatically affect your entire website.

:::note

This tutorial has been tested with the following package versions in Node.js v24.16.0:

- `react@19.2.7`
- `react-dom@19.2.7`
- `@storyblok/react@5.0.0`
- `tailwind@4.3.1`
- `preline@4.2.0`

:::

## General steps

In this tutorial you will perform the following steps:

1. Create a new Storyblok space from a Blueprint
1. Create the content model
1. Set up additional application dependencies
1. Create React components in your application
1. Add content in Storyblok
1. Load navbar and footer configuration

Refer to the following sections for step-by-step instructions.

## 1. Create a new Storyblok space from a Blueprint

First, create a new blank Storyblok space based on the [Storyblok Core Blueprint for React](https://github.com/storyblok/blueprint-core-react). For instructions on launching a new project using a Storyblok Blueprint, refer to [Blueprints](https://www.storyblok.com/docs/concepts/blueprints).

After setting up your Storyblok space and GitHub repository, clone your repository locally and set up your access token and region.

### Set up access token

In the root of the project, create a copy of the `.env.example` file and rename it to `.env`.

In Storyblok, copy your space's preview [access token](https://www.storyblok.com/docs/concepts/access-tokens) from **Settings** > **Access Tokens** and then enter your access token in the `.env` file:

```ini title=".env" "<REPLACE_WITH_YOUR_TOKEN>"
STORYBLOK_DELIVERY_API_TOKEN=<REPLACE_WITH_YOUR_TOKEN>
STORYBLOK_REGION=<REPLACE_WITH_SPACE_REGION>
```

### Enter your space region

Still in the `.env` file, enter a region value in `STORYBLOK_REGION` or delete the entire line if your space region is the EU (the default region value is defined as `eu` in `src/main.jsx`).

```ini title=".env" "<REPLACE_WITH_SPACE_REGION>"
STORYBLOK_DELIVERY_API_TOKEN=<REPLACE_WITH_YOUR_TOKEN>
STORYBLOK_REGION=<REPLACE_WITH_SPACE_REGION>
```

For more information, refer to [The region parameter](https://www.storyblok.com/docs/libraries/js/js-sdk#the-region-parameter) in the JavaScript SDK documentation.

## 2. Create the content model

Customize the Storyblok content model according to the use case of this tutorial.

In the [block library](https://www.storyblok.com/docs/concepts/blocks), create the following nestable blocks:

- `nav_entry`
  - `title`: [text field](https://www.storyblok.com/docs/concepts/fields#text)
  - `link`: [link field](https://www.storyblok.com/docs/concepts/fields#link)
- `nav_submenu`
  - `title`: [text field](https://www.storyblok.com/docs/concepts/fields#text)
  - `items`: [blocks field](https://www.storyblok.com/docs/concepts/fields#blocks), configured to only allow blocks `nav_entry` and `nav_submenu`
- `footer_entry`
  - `title`: [text field](https://www.storyblok.com/docs/concepts/fields#text)
  - `link`: [link field](https://www.storyblok.com/docs/concepts/fields#link)
- `footer_submenu`
  - `title`: [text field](https://www.storyblok.com/docs/concepts/fields#text)
  - `items`: [blocks field](https://www.storyblok.com/docs/concepts/fields#blocks), configured to only allow blocks `footer_entry` and `footer_submenu`

Still in the block library, create the `config` content type:

- `config`
  - `nav_items`: [blocks field](https://www.storyblok.com/docs/concepts/fields#blocks), configured to only allow blocks `nav_entry` and `nav_submenu`.
  - `footer_items`: [blocks field](https://www.storyblok.com/docs/concepts/fields#blocks), configured to only allow blocks `footer_entry` and `footer_submenu`.

Later, you will create a [story](https://www.storyblok.com/docs/manuals/stories) based on this content type to hold the global navbar and footer configuration.

:::tip
If you're unfamiliar with content modeling in Storyblok, refer to [Blocks](https://www.storyblok.com/docs/concepts/blocks) and [Fields](https://www.storyblok.com/docs/concepts/fields) for more information.
:::

The `NavEntry` and `NavSubmenu` components support both top-level and child-level entries. The same applies to the footer components `FooterEntry` and `FooterSubmenu`. Since different HTML elements and CSS classes are required for top-level entries, a prop named `topLevelNav` on these four React components allows us to specify which navbar/footer elements are at the top level.

:::tip
While you could use different elements (and blocks) for the two different contexts, this strategy reduces the number of React components and the number of different blocks in your Storyblok space.
:::

## 5. Add content in Storyblok

Before proceeding, make sure that you have started the local web server for your React app using the following command:

```bash
npm run dev
```

Next, update your preview URL in Storyblock so that it points to your local web server (`https://localhost:5173/` by default). 

1. Open **Settings** → **Visual Editor**.
1. Under **Preview URLs**, enter `Dev` in **Name** and your preview URL in **Location**.
1. Select **Save**.
1. In the Visual Editor, change the preview URL to the development URL you just defined.

In Storyblok, create a new story based on the `config` content type you created earlier.

1. Switch to the **Content** tab.
1. Select **Create new content** → **Story**.
1. In **Name**, enter `Config`.
1. In **Content type**, select _Config_.
1. Select **Create**.

Next, add some entries to the navbar.

1. In the **Content** tab, select the Config story.
1. If necessary, open the **Edit** sidebar.
1. Under **Nav Items**, select **Add Block**.
1. Select the desired block:
   - To add a basic entry with a title and a link, select the **Nav Entry** block. You can link to external URLs or to existing Storyblok stories.
   - To add a new navbar menu, select **Nav Submenu**. You will need to provide an entry title and the menu items. To add another level of submenus, add a Nav Submenu as a nested block.
1. Expand the block details and configure the fields as needed.
1. Add more navigation entries. Use different blocks (Nav Entry or Nav Submenu) at different levels to explore all the possibilities of the navbar.
1. Select **Save**.


To finalize the configuration, add some entries to the footer. Add Footer Entry or Footer Submenu blocks to the **Footer Items** field. The configuration is similar to the navbar configuration, but the footer only supports one level of submenus instead of nested submenus.

:::note
You must load the navbar and footer configuration in your application before you can see any changes in the Visual Editor. You will do this in the next section.
:::
